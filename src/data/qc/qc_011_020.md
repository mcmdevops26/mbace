## scenario_020: FLAG
**Issues:**
1. **Step 2 math is visibly broken mid-calculation.** Step 2 contains "wait, recalculate properly:" in the middle of the analysis — an unedited revision artifact left in the published scenario. This is unprofessional and confusing. A student reading the stepByStep will hit "wait, recalculate properly" and be disoriented. The final calculation in step 2 is also wrong in presentation: "New gross profit = $120 - ($110 × 52%) - $8 = $120 - $57.20 - $8 = $54.80" — this formula structure is mathematically correct as presented but the setup using "(110 × 52%)" as COGS is inconsistently derived from the 48% gross margin (implying 52% COGS margin), which is correct but not explicitly stated.
2. **Typo in meceBlocks.** Sub-issue reads "how many margin points does Niek lose?" — "Niek" should be "Nike."
3. **Key term definition has an error.** The definition for "Price elasticity of demand" states: "values above -1.0 are 'elastic' (price-sensitive) and values below -1.0 are 'inelastic.'" This is backwards. Elasticity of -1.4 is more elastic (price-sensitive) than -0.5. The correct statement is: absolute values greater than 1.0 are elastic; absolute values less than 1.0 are inelastic. As written, "above -1.0" would mean values like -0.5 (which is actually inelastic), making the definition the reverse of the truth. This is a factual error in a key term definition that a student will memorize.
4. **Step 3 math is correct but complex to follow.** The $20 increase model correctly calculates: COGS = $57.20 + $8 = $65.20, gross profit = $130 - $65.20 = $64.80, volume = 11.2M, total GP = $725.8M vs. $792M baseline ✓. These figures check out.
5. **Step 4's $5 increase recommendation is internally consistent with sample response.** Both recommend "$5-7 increase." The math ($699M vs. $792M vs. $672M) is correctly done in step 4.
6. **"Niek" typo should be an easy fix.** This appears in the meceBlocks, not the sampleResponse.
7. **The overall scenario is the most math-intensive in the set and is correctly designed.** The core insight — that -1.4 elasticity makes the $20 increase financially destructive, and the recommendation for a smaller increase — is analytically correct and teachable.
8. **Framework fit is correct.** Breakeven Analysis is the centerpiece and drives the recommendation. Pricing Framework and 4P's wrap the quantitative core appropriately.

**Worst offender:** The price elasticity key term definition is factually inverted. A student who memorizes "values above -1.0 are elastic" will have incorrect understanding of a fundamental economics concept going into interviews.
